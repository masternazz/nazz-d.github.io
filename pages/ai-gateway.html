AI Gateway

Node.js service running on a Proxmox LXC that takes Telegram messages and emails, routes them to Claude, Gemini CLI, or Codex, and sends the reply back. Supports git ops inline, streams Claude and Codex replies live, and keeps shared memory across sessions.

What It Does

  • Receives Telegram messages via webhook and emails via IMAP polling, routes each to the selected AI CLI, and sends the response back.
  • Supports three AI engines — Claude, Gemini CLI, and OpenAI Codex — switchable per message with a prefix (gemini:, codex:, default is Claude).
  • Parses git operation phrases (pull chess, push homelab-docs) and runs them against SSH-authenticated repos before passing context to the AI.
  • Streams Claude and Codex responses live to Telegram — the reply message updates every 1.5s as output arrives.
  • Maintains per-channel conversation history and a shared persistent memory file all AIs read on every request.
  • Restricts access to a single allowed Telegram user ID and a domain allowlist for email.

Architecture

  • Runtime: Node.js 20 LTS on a Proxmox LXC, exposed via Cloudflare Tunnel. Express handles the Telegram webhook endpoint.
  • AI engines: Each CLI is spawned as a child process. Claude and Codex receive the prompt over stdin. Gemini takes it as a direct -p argument with --yolo to auto-approve agentic tool calls.
  • Git: simple-git with SSH key auth via GIT_SSH_COMMAND. Per-repo clone locks prevent race conditions when multiple operations target the same repo simultaneously.
  • Memory: A memory.md flat file prepended to every prompt. Per-channel history stored as JSON, compacted automatically when it grows past 20 entries.
  • Email: IMAP polling with imapflow. Replies sent via SMTP. Domains outside the allowlist are silently dropped.

Parsing

Smart Message Parser

Each incoming message is parsed for engine prefix, verbose flag, and git operation phrases. Git phrases (validated against the configured repo list) are extracted into structured { op, target } objects and stripped from the AI prompt — so the AI never sees "pull chess" but does get the actual git log output as context.

Gemini

Taming Agentic Behavior

Gemini CLI's agentic mode silently hangs when it detects repo-related context and tries to call shell tools that fail without output. Fixed with two changes: --yolo auto-approves all tool calls so nothing blocks, and the prompt is passed as a direct -p argument instead of stdin — which suppresses the agentic trigger path entirely.

Git

Concurrent Clone Safety

When a message triggers both a git pull and a log fetch on an uncached repo, two concurrent ensureCloned calls race to clone the same directory. Fixed with an in-memory promise lock — the second caller awaits the first clone's promise instead of starting a new one, then returns as soon as the directory exists.

Employer-Relevant Skills

This project required working at the edge of what AI CLIs actually support: diagnosing silent hang behavior in Gemini's agentic mode, fixing race conditions in concurrent git operations, and debugging SSH auth failures in simple-git's subprocess layer. The result is a system that routes messages across three AI runtimes with real git integration, streaming output, and persistent context — all deployed on self-managed infrastructure.
