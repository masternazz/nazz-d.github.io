Project | Networking & Security

Enterprise VLAN Segmentation

Redesigned a flat homelab network from a single broadcast domain to a fully segmented 10-VLAN production architecture — each VLAN with a defined security posture, explicit OPNsense firewall rules, and zero unnecessary cross-VLAN exposure. 30+ services migrated, 80+ firewall rules enforced.

OPNsense 802.1Q VLANs Cisco SG300-20 Netgear GS728TP Firewall Design Default-Deny DNS Migration Proxmox

What I Built

  • 10-VLAN Design: Segmented the entire lab across 10 VLANs on a role-based private addressing scheme — management, game servers, DMZ, services, storage, servers, IoT, testing, guest, and OOB management each isolated at the firewall.
  • OPNsense Default-Deny: Enforced 80+ explicit allow rules across all VLAN interfaces — if there is no rule, traffic is blocked. Every cross-VLAN path that needs to work is documented as a firewall rule.
  • Switch Hardware Swap: Replaced legacy Cisco Catalyst 3750V2 (100 Mbps) switches mid-migration with Cisco SG300-20 core + Netgear GS728TP PoE+ access. Re-trunked all VLANs from scratch.
  • Proxmox Quorum Recovery: Restored 3/3 corosync quorum after the switch swap left VLAN 10 absent from Proxmox node ports — identified via LLDP, fixed via SG300 trunk config.
  • Bulk Service Migration: Moved 30+ VMs and LXC containers to correct VLANs, updated DNS host overrides via OPNsense API, reconfigured NPMplus reverse proxy backends, and updated NAT policy.
  • DMZ Architecture: cloudflared and NPMplus isolated in a dedicated DMZ VLAN. NPMplus proxies to approved backend services only — no DMZ host can initiate connections to management or storage.

Security Design Decisions

  • Management VLAN: Admin PC, Proxmox nodes, switches, and OPNsense management interface. The only VLAN allowed to initiate connections broadly. No service VLAN can reach management without an explicit rule.
  • DMZ VLAN: cloudflared (Cloudflare Tunnel) and NPMplus (reverse proxy) are the only hosts. cloudflared is permitted to reach NPMplus only. NPMplus is permitted to reach approved backend services. Nothing in DMZ initiates to management.
  • Storage VLAN: NAS and backup targets cannot initiate outbound internet connections. Management and approved virtualization hosts are the only sources permitted to reach storage services.
  • Guest / IoT VLANs: Internet-only policy with private-network blocking. Guest uses OPNsense captive portal. IoT devices cannot reach infrastructure VLANs.
  • Wazuh log collection: Each VLAN has scoped rules allowing log and agent traffic to the monitoring stack. Security monitoring works without general cross-VLAN access.
Addressing

Structured IP Scheme

Every VLAN follows a consistent private addressing pattern with separate ranges for gateways, infrastructure, services, DHCP clients, and spare capacity. The exact subnet plan stays private, but the public-safe takeaway is that each role has a predictable network location and a clear firewall boundary.

Enforcement

OPNsense Firewall Policy

OPNsense applies rules per VLAN interface. Default-deny means any unlisted traffic is silently dropped. Each new service requires a conscious decision: which VLAN source, which destination IP and port, and why. No implicit trust between any two VLANs — not even adjacent ones.

Public Ingress

DMZ-Only Internet Exposure

Web-facing services use Cloudflare Tunnel and NPMplus in VLAN 7 where possible. Game servers, mail flow, VPN/relay traffic, and remote-support services use documented operational WAN forwards. Management ports, Proxmox UI, and direct SSH stay off the public WAN.
